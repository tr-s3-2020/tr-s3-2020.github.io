Chap 1. Basic concepts

2023-04-10 Reinforcement Learning

A grid-world example

State and Action

State

status of the agent (location in the grid example)
state space: S={s1,…s9}S = \{s_1, \dots s_9\}S={s1​,…s9​}

Action

action to take, for each state. (move direction in the grid example)
action space A(si)={a1…a5}\mathcal{A}(s_i) = \{a_1 \dots a_5\}A(si​)={a1​…a5​}. Noticing that here AAA is related to sis_isi​

  • different states have different action spaces
  • action space can be compressed under certain situations (i.e. on boundary states, some directions are forbidden)
  • RL's purpose is to evaluate actions

State Transition

\ce{s_1 ->[a_1] s_2}

  • the transition process can be concluded in a table, called tabular representation
  • the transition process can also be described in math.
    • p(si∣si,aj)=constp(s_i|s_i,a_j)=constp(si​∣si​,aj​)=const

Policy

A policy tells the agent which actions to take at each state.
Use a conditional probability distribution function π(a∣s)\pi(a|s)π(a∣s)
We can still use a tabular representation to represent policies. (we have another representation called parameterized functions)

Idea: What's the difference between state transition and policy

State transition and policy can both be represented in conditional possibilities, while they are different.
State transition is a fact. It tells us, for certain state, if you take certain action, what would happen. It gives us a picture of the fact. No decision is involved.
Policy is a decision. It tells us to what extend do the agent prone to certain action.

Reward

One of the most unique concepts in RL.

  • Designing appropriate rewards is an important step of RL.
  • Bellman optimality equation is used to define the optimality of policies (with optimal reward).

Reward transition

the process of getting a reward after taking an action

  • Use tabular representation to represent it.
  • use math representation: p(r∣s,a)=constp(r|s,a)=constp(r∣s,a)=const
  • reward transition is also a fact, not a decision

Q: Whether we should use action or next state to define reward?
p(r∣s,a)p(r|s,a)p(r∣s,a) or p(r∣s,s′)p(r|s,s')p(r∣s,s′)
A: equivalent in deterministic situation, not in stochastic ones. Should use action instead of next state since we want to reward or punish action instead of next state. Also, for the same destination state, there would be different actions to reach it with different rewards. (过程很重要)

Trajectory, return and episode

Trajectory

a state-action-reward chain

Return

For each trajectory, return is the sum of all rewards
immediate reward + delayed reaward

discounted reward: avoid diverging, define short-sighted and long-sighted

Episode

trajectory which stops at a terminal state (finite trajectory)

Task

a task consists of several episodes
episodic task (finite) & continuous task (infinite)

unify all tasks to continuous task with discounted rewards

Markov decision process

sets, model, policy, markov property

Sets

  • state set SSS
  • action set A(s)\mathcal{A(s)}A(s)
  • reward set R(s,a)\mathcal{R(s,a)}R(s,a)

Model (fact)

  • state transition probability p(s′∣s,a)p(s'|s,a)p(s′∣s,a)

  • reward transition probability p(r∣s,a)p(r|s,a)p(r∣s,a)

  • model-free / model-based RL algorithms

  • stationary(time-invariant) or nonstationary(time-variant)

Policy (decision)

  • π(a∣s)\pi(a|s)π(a∣s)
